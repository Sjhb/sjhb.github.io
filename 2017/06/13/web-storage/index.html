Web Storage 浏览器端的数据缓存方案

王欢 6月 13, 2017
  • 分享到微博
  • 分享到 Twitter
  • 分享到 Facebook
  • 分享到 Google+

概述

Web storage 这个API是在HTML5中新提供的，用来在浏览器端存储数据。在web storage出现之前，多通过cookie来实现在浏览器端存储数据。
Web storage分为两类：sessionStorage和localStorage，它们都能且只能读写当前域的数据，主要区别在于localStorage存储的数据不会过期。具体为：sessionStorage保存的数据具有时间限制，当会话结束后（例如关闭窗口），在本地存储的数据会被清空；localStorage保存的数据会长期存在浏览器端，下一次访问相同的网页时，可以直接读取保存的数据。

经过测试，虽然localStorage是保存在本地的，但是不同的浏览器保存的数据是独立的，也就是说在chrome中保存的localStorage在FireFox中是取不到的。

目前，web storage的存储上限根据浏览器的不同也各有不同，Chrome是2.5MB，Firefox和Opera是5MB，IE是10MB。
在实际的开发过程中，可以通过检查window对象是否包含sessionStorage和localStorage属性的方式，来确定浏览器是否支持这两个对象。

function checkStorageSupport() {

  // sessionStorage
  if (window.sessionStorage) {
    return true;
  } else {
    return false;
  }

  // localStorage
  if (window.localStorage) {
    return true;
  } else {
    return false;
  }
}

其实，在调用sessionStorage和localStorage时都会创建并返回Storage对象，这两个Storage对象是独立的。调用这个Storage对象的方法(setItem,clrea等)可以实现对数据的操作。更确切的说，在支持的浏览器中 Window 对象实现了 WindowLocalStorage 和 WindowSessionStorage 对象并挂在其 localStorage 和 sessionStorage 属性下。


与cookie的比较

  • cookie是http协议规范的一部分，可以使用cookie来进行服务器和浏览器之间的交互，例如，服务器可以通过HTTP的Set-Cookie消息头向浏览器发送保存一个cookie（关于cookie的详细信息可以点击查看 cookie）。浏览器脚本也可以向本地读写cookie。而web storage只是为了在浏览器端保存数据而生。
  • 简单的来说，cookie就是一小段保存在本地的文本信息，在保存的时候是以一段字符传的形式，而web storage则是以键值对的形式保存。
  • cookie自身是轻量级的，大小和数量都受到了严格的限制，例如各个cookie的大小不能超过4kb，否则就会被裁掉。web storage在概念上与cookie相似，但是web storage更加适合于存储较大体积的数据。
  • cookie使用起来更为复杂，需要开发者自己封装setCookie，getCookie，而web storage提供了setItem，getItem等方法，使用起来更为方便快捷。
  • cookie和web storage 一样都会受到同域的限制。

具体操作

sessionStorage和loacalStroage都是以文本格式保存保存数据，保存的数据以键值对的形式存在，它们具有相同的操作方法，具体如下。

写入数据

存入数据时，使用setItem(key,value)方法，该方法接受两个参数，第一个是键名，第二个是数据值。

sessionStorage.setItem("key","value");

localStorage.setItem("key","value");

读取数据

getItem(key)接收键名作为它唯一的参数,返回具体的数据值，如果没有对应的值就会返回null。

var valueSession = sessionStorage.getItem("key");

var valueLocal = localStorage.getItem("key");

清除数据

removeItem(key)方法接收一个参数：键名，用来清除指定的数据。

sessionStorage.removeItem("key");

localStorage.removeItem("key");

clear()方法用来清除所有数据，没有参数。经测试无返回值，即undefined。

sessionStorage.clear();

localStorage.clear();

遍历操作

sessionStorage和localStorage具有length属性，表示键值对数量。key(index)方法用来取出指定位置的键值,计数从零开始。

sessionStorage.setItem("s","sessionstorage");
localStorage.setItem("l","localStorage");

consle.log(sessionStorage.length);
consle.log(sessionStorage.key(0));
consle.log(localStorage.length);
consle.log(localStorage.key(0));
/*输出结果：
1
sessionStorage
1
localStorage
*/

利用length属性和key方法，可以遍历所有的键。

for(let i = 0; i < localStorage.length; i++){
    console.log(localStorage.key(i));
}

for(let i = 0; i < sessionStorage.length; i++){
    console.log(sessionStorage.key(i));
}

其他操作方法：. 操作和 [ ] 操作

web storage可以通过setItem和getItem方便地存取数据，也可以像操作对象一样操作其中的数据，因为它本身就是一个对象。

sessionStorage.key1="s_key1";
sessionStorage["key2"]="s_key2";

console.log(sessionStroage["key1"]+"\t"+sessionStorage.key2);
//输出结果：s_key1    s_key2

localStorage.key1="l_key1";
localStorage["key2"]="l_key2";

console.log(localStroage["key1"]+"\t"+localStorage.key2);
//输出结果l_key1    l_key2

个人认为，通过操作属性的方法操作数据不够规范，不推荐使用。

storage事件

可以通过window.addEventListener这个方法来像顶层对象window注册storage监听器，并指定事件的回调函数，当数据发生变化的时候就会触发这个回调函数。注意，回调函数不能返回任何结果，具体关于window.addEventListener的详情可参考文档：EventTarget.addEventListener() - Web APIs | MDN

window.addEventListener("storage",onStorageChange);

function onStorageChange(e) {
     console.log(e.key);
}

在上面的例子中，回调函数接受了一个event对象作为参数，这个参数的key属性保存这变化的键名。除了key，event对象还有其他属性。

属性 类型 描述
key String 变化的键名。操作包括：添加、移除、修改
oldValue Any 更新前的值。如果该键为新增加，则这个属性为null。
newValue Any 更新后的值。如果该键被删除，则这个属性为null。
url String 原始触发storage事件的那个网页的网址。

特别注意:除了IE浏览器之外，其他浏览器的Sotrage事件不在导致数据变化的当前页面触发。如果浏览器同时打开一个域名下面的多个页面，当其中的一个页面改变sessionStorage或localStorage的数据时，其他所有页面的storage事件会被触发，而原始页面并不触发storage事件。可以通过这种机制，实现多个窗口之间的通信。IE浏览器会在当前页面触发storage事件。

浏览器兼容情况

Web storage兼容情况在HTML5的众多新特性中算是比较好的，除了IE7，IE6不支持之外，其它的浏览器基本都支持,IE8以下的版本可以使用IE userData,具体可以参考 userDataBehavior。
